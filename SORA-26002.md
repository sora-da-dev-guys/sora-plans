#### API function to add in `src/apis/sbb-api/apis.ts`

```ts
// GET /quotations/table-data
export const getQuotationTableDataApi = async (
  query: GetQuotationTableDataQuery
): Promise<IQuotationTableData> => {
  const queryString = buildQueryString(query as Record<string, string | undefined>)
  const response = await fetch(`${SBB_API_URL}/quotations/table-data${queryString}`, {
    method: 'GET',
    headers: getAuthHeaders(),
  })
  if (!response.ok) {
    if (response.status === 401) handleUnauthorized()
    const error = await response.json().catch(() => ({}))
    throw new Error(
      (error as { message?: string }).message || 'Failed to fetch quotation table data'
    )
  }
  return response.json()
}
```

#### Hook `src/hooks/sbb-api/useQuotations.ts`

```ts
import { useQuery } from '@tanstack/react-query'
import { getQuotationTableDataApi } from '@/apis/sbb-api'
import type { GetQuotationTableDataQuery } from '@/apis/sbb-api'

export const useQuotationTableData = (query: GetQuotationTableDataQuery) => {
  return useQuery({
    queryKey: ['quotations', 'table-data', query],
    queryFn: () => getQuotationTableDataApi(query),
  })
}
```

Export from `src/hooks/sbb-api/index.ts`.

#### Page `src/pages/quotations/QuotationListPage.tsx`

Columns (based on SBB spec):
- **Status** — status chip with colour by status id
- **Sale Order** — icon indicator: green doc icon if `activeSaleOrderAmount > 0`, disabled if 0; tooltip showing count
- **Quotation Code** — link to `/quotations/:id` (detail page — not in this plan, just the link)
- **Created Date** — formatted date
- **Created By** — `recCreatedBy`
- **Customer Name** — `customer.nameLocal`
- **Phone** — `customer.phoneNumber`
- **Payment Method** — `paymentMethod.name`
- **Total** — formatted number (e.g. `฿1,234.00`)

Filters:
- Status filter (multi-select) — dynamically built from loaded data (distinct statuses)
- Other: "No sale order" checkbox, "Need to follow" checkbox
- Date range filter for `recCreatedAt` (startDate / endDate in URL params)

Pagination: same pattern as `CustomerListPage` (URL params, prev/next, go-to page).

Status chip colours:
```
Pending (1)       → yellow  (bg-yellow-100 text-yellow-800)
Approve (2)       → green   (bg-green-100 text-green-800)
Cancelled (3)     → gray    (bg-gray-100 text-gray-800)
Rejected (4)      → red     (bg-red-100 text-red-800)
PendingReview (5) → blue    (bg-blue-100 text-blue-800)
```
