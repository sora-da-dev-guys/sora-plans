**ID:** SORA-26004
**Repo:** smartbackoffice
**Date:** 2026-03-14

## Requirements Summary

- Improve **Product List Page** to match `smartbestbuys-backoffice` spec
- Add product type tabs (Product / Service / Internal / Compound / Asset)
- Add grade filter, pending verify, and request changed checkboxes
- Add richer stock columns (Incoming, SO Reserved, Reserved, Ready to Sell)
- Add image thumbnail in table rows
- Extend types to support new fields and query params

## Implementation Notes

**Current state:** Search by name/code, status filter, table (code, name, unit, category, stock, price, status), pagination. Located at `src/pages/ProductListPage.tsx` — no relocation.

### Type additions in `src/apis/sbb-api/types.ts`

#### New constants

```ts
// Product type constants (no enums)
export const PRODUCT_TYPE = {
  Product: 1,
  Service: 2,
  InternalProduct: 3,
  CompoundProduct: 4,
  Asset: 5,
} as const

export type ProductTypeValue = (typeof PRODUCT_TYPE)[keyof typeof PRODUCT_TYPE]

// Product mode constants
export const PRODUCT_MODE = {
  Available: 0,
  Archive: 1,
} as const

export type ProductModeValue = (typeof PRODUCT_MODE)[keyof typeof PRODUCT_MODE]
```

#### Extended `IProductTableData_Data`

```ts
export interface IProductTableData_Data {
  id: number
  code: string
  nameLocal: string
  nameEn?: string
  unitLocal: string
  unitEn?: string
  category?: BaseData
  grade?: string              // ➕ A/B/C/D/F
  imageUrl?: string           // ➕ product thumbnail URL
  stockAmount: number
  incomingAmount?: number     // ➕ stock being received
  reservedAmount?: number     // ➕ stock reserved for shipping
  soReservedAmount?: number   // ➕ stock reserved by open SOs
  withdrawnAmount?: number    // ➕ stock currently borrowed
  saleAmount?: number         // ➕ total sold; 0 = dead stock
  price: number
  status: BaseData
}
```

#### Extended `GetProductTableDataSearchParams`

```ts
export interface GetProductTableDataSearchParams {
  search?: string
  status?: string
  typeIds?: string              // ➕ selected product type id
  gradeList?: string            // ➕ comma-separated grade values
  mode?: string                 // ➕ "0"=Available / "1"=Archive
  isPendingVerified?: string    // ➕ "true" to filter pending verify
  isRequestChangedData?: string // ➕ "true" to filter request changed
}
```

### Updated table columns

| Column | Notes |
|--------|-------|
| Thumbnail | Small `<img>` ~40×40px; fallback to placeholder icon when `imageUrl` is null |
| Code + Name | Existing — keep (code link, nameLocal, nameEn) |
| Unit | Existing — keep |
| Category | Existing — keep |
| Stock | Existing — keep (total `stockAmount`) |
| Incoming | ➕ `incomingAmount`; hidden when Service tab |
| SO Reserved | ➕ `soReservedAmount`; always shown |
| Reserved | ➕ `reservedAmount`; hidden when Service tab |
| Ready to Sell | ➕ `max(stockAmount - soReservedAmount, 0)`; hidden when Service tab |
| Price | Existing — keep |
| Status | Existing — keep |

**Dead stock indicator:** Show small `Dead Stock` badge when `saleAmount === 0`.
