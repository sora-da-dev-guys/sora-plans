# Plan: User List Page

**ID:** SORA-26005
**Repo:** smartbackoffice
**Date:** 2026-03-17

## Requirements Summary

- Build a **new** User List page at `/users` in **smartbackoffice** (React 19, Vite, TanStack Query, Jotai, Tailwind — no MUI)
- Reference spec from `smartbestbuys-backoffice`: table showing User ID, Full Name, Email, Role/Position, Status, Department, Last Login

- Action buttons: View Profile (link), Create New User (permission-gated)
- Register route in `App.tsx` and add nav item to sidebar in `BackofficeLayout`

## Implementation Notes

### What Exists vs. What's Missing

**Already available in `smartbackoffice`:**
- `IUserListData` type (`id`, `username`, `firstName`, `lastName`, `nickName`, `imageUrl`, `phoneNumber`, `email`) — in `src/apis/sbb-api/types.ts`
- `getUserListApi` → `GET /data/v1/user-data` — in `src/apis/sbb-api/apis.ts`
- `useUserList` hook (TanStack Query, staleTime 10min) — in `src/hooks/sbb-api/useSaleOrders.ts` (misplaced, reuse as-is)
- Permission atom: `permissionAtom` from `src/atoms/auth.ts` → `hr.user.read/write/delete`

**Missing from `IUserListData` vs. spec:**
- `enabled` / status (Active/Inactive)
- `role`, `department`, `lastLogin`

> **Decision:** These fields don't exist in the current API response (`/data/v1/user-data`). The spec references `/users/v1` as the proper endpoint. Create a new API type `IUserTableData` and a new API call `GET /users/v1` that should return status + full user data. Add this as a new function in `src/apis/sbb-api/apis.ts`. If the backend endpoint isn't ready, fall back to `getUserListApi` and render `–` for missing fields.

### New API Function

```typescript
// To add to src/apis/sbb-api/apis.ts
export const getUserTableDataApi = async (): Promise<IUserTableData[]> => {
  const response = await fetch(`${SBB_API_URL}/users/v1`, {
    method: 'GET',
    headers: getAuthHeaders(),
  })
  if (!response.ok) {
    if (response.status === 401) handleUnauthorized()
    const error = await response.json().catch(() => ({}))
    throw new Error((error as { message?: string }).message || 'Failed to fetch users')
  }
  return response.json()
}
```

### New Hook

```typescript
// New file: src/hooks/sbb-api/useUsers.ts
export const useUserTableData = () => {
  return useQuery({
    queryKey: ['user-table-data'],
    queryFn: getUserTableDataApi,
    staleTime: 1000 * 60 * 5,
  })
}
```

Export via `src/hooks/sbb-api/index.ts`.

### Page Pattern

Follow `CustomerListPage.tsx` exactly:
- URL-based filter state via `useSearchParams`
- `useDebounce` on search input
- Client-side filter + slice for pagination (page size: 10)
- `PageHeader` with `right` prop for Create button
- `Table` + `PaginationBar` from existing components
- `Badge` for status (green = Active, gray = Inactive)
- `LoadingState`, `ErrorBanner`, `EmptyState` from feedback components

### Permission Gate

```typescript
const [permission] = useAtom(permissionAtom)
const canWrite = permission?.hr.user.write ?? false
const canRead  = permission?.hr.user.read  ?? false
```

- "Create User" button: disabled + tooltip when `!canWrite`
- Row click → `/users/{id}` (read-gated, but list itself always visible)

### Routing

Add to `App.tsx`:
```tsx
{ element: <UserListPage />, path: 'users' }
```

Add to sidebar (`BackofficeLayout.tsx`):
```tsx
<SideNavItem to="/users" icon={<Users className="size-4" />} label="Users" />
```
